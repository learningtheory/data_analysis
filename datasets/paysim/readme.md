上下文
缺乏关于金融服务的公开数据集，特别是在新兴的移动货币交易领域。金融数据集对于许多研究人员非常重要，尤其对于我们在欺诈检测领域进行研究。部分问题是金融交易的内在私人性质，导致没有公开可用的数据集。

我们提供了一个使用名为PaySim的模拟器生成的综合数据集，作为解决此类问题的一种方法。PaySim使用来自私人数据集的汇总数据生成类似于交易正常操作的合成数据集，并注入恶意行为以便稍后评估欺诈检测方法的性能。

内容
PaySim根据从非洲国家实施的移动货币服务的一个月财务日志中提取的真实交易样本来模拟移动货币交易。原始日志由一家跨国公司提供，该公司是目前在全球14个以上国家运营的移动金融服务提供商。

这个合成数据集缩小了原始数据集的1/4，并且仅为Kaggle创建。

头
这是一个带有标题说明的行的示例：

1，付款，1060.31，C429214117,1089.0,28.69，M1591654462,0.0,0.0,0,0

步骤 - 在现实世界中映射一个时间单位。在这种情况下，1步是1个小时的时间。总步骤744（模拟30天）。

类型 - 现金，现金，支付，付款和转让。

金额 - 以当地货币交易的金额。

nameOrig - 开始交易的客户

oldbalanceOrg - 交易前的初始余额

newbalanceOrig - 交易后的新余额

nameDest - 交易接收方的客户

oldbalanceDest - 交易前的初始余额收款人。请注意，没有以M（Merchants）开头的客户的信息。

newbalanceDest - 交易后的新余额收款人。请注意，没有以M（Merchants）开头的客户的信息。

isFraud - 这是由模拟中的欺诈代理进行的交易。在这个特定的数据集中，代理人的欺诈行为旨在通过控制或客户账户获利，并尝试通过转移到另一个账户清空资金，然后兑现出系统。

isFlaggedFraud - 商业模式旨在控制从一个账户到另一个账户的大规模转账，并标记非法企图。此数据集中的非法尝试是尝试在单个事务中转移超过200.000。

过去的研究
有5个类似的文件包含5个不同场景的运行。这些文件在我的博士论文第7章（博士论文可在http://urn.kb.se/resolve?urn=urn:nbn:se:bth-12932）中得到更好的解释。

我们使用744个随机种子对PaySim进行了多次运行，代表了实时一个月的每个小时，与原始日志相匹配。每次运行需要大约45分钟的时间，配备16GB内存的i7英特尔处理器。运行的最终结果包含大约2400万条财务记录，分为五类：现金，现金，借方，付款和转账。
